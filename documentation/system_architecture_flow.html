2. Authentication & Role-Based Access Flow

This flow shows how users authenticate and are routed to their respective dashboards based on their role.

        sequenceDiagram
            participant User
            participant Frontend
            participant Backend
            participant DB
            
            User->>Frontend: Enter Credentials (Email & Password)
            Frontend->>Backend: POST /api/v1/auth/login
            Backend->>DB: Query User by Email
            DB-->>Backend: Return User Record
            Backend->>Backend: Verify Password Hash
            
            alt Invalid Credentials
                Backend-->>Frontend: 401 Unauthorized
                Frontend-->>User: Show Error Message
            else Valid Credentials
                Backend->>Backend: Generate JWT Token
                Backend-->>Frontend: 200 OK (Token + User Data)
                Frontend->>Frontend: Store Token in localStorage
                
                Note over Frontend: Routing logic based on Role
                
                alt Role == SUPER_ADMIN / ORG_ADMIN
                    Frontend->>User: Redirect to /admin/dashboard
                else Role == HR_MANAGER
                    Frontend->>User: Redirect to /hr/dashboard
                else Role == FINANCE
                    Frontend->>User: Redirect to /finance/dashboard
                else Role == MANAGER / TEAM_LEAD
                    Frontend->>User: Redirect to /manager/dashboard
                else Role == EMPLOYEE
                    Frontend->>User: Redirect to /employee/dashboard
                end
            end

3. Employee Onboarding Flow

This flow illustrates the sequence of actions when a new employee is hired and joins the organization.

        graph TD
            classDef hr fill:#ec4899,stroke:#be185d,stroke-width:2px,color:#fff
            classDef sys fill:#10b981,stroke:#047857,stroke-width:2px,color:#fff
            classDef emp fill:#3b82f6,stroke:#1d4ed8,stroke-width:2px,color:#fff

            Start([HR Creates Employee]) --> |HR Portal| SysCreate(System Creates Profile & User Account)
            SysCreate --> SysEmail(System Sends Default Credentials to Employee)
            
            SysEmail --> EmpLogin([Employee Logs In])
            EmpLogin --> Wizard{Onboarding Completed?}
            
            Wizard -- No --> ShowWizard(Show Onboarding Wizard)
            ShowWizard --> UpdateProfile(Employee Completes Profile)
            UpdateProfile --> UploadDocs(Employee Uploads Docs)
            UploadDocs --> AckPolicies(Employee Acknowledges Policies)
            AckPolicies --> DoneWizard(System Marks Onboarding = true)
            DoneWizard --> Dash([Access Standard Dashboard])
            
            Wizard -- Yes --> Dash
            
            class Start hr
            class SysCreate,SysEmail,Wizard,ShowWizard,DoneWizard sys
            class EmpLogin,UpdateProfile,UploadDocs,AckPolicies,Dash emp

4. Time, Leave, and Payroll Data Flow

This diagram shows how daily time tracking and leave requests eventually feed into the monthly payroll processing.

        graph LR
            classDef emp fill:#3b82f6,stroke:#1d4ed8,stroke-width:2px,color:#fff
            classDef mgr fill:#f59e0b,stroke:#b45309,stroke-width:2px,color:#fff
            classDef sys fill:#10b981,stroke:#047857,stroke-width:2px,color:#fff
            classDef fin fill:#8b5cf6,stroke:#6d28d9,stroke-width:2px,color:#fff

            subgraph Daily Tracking
                Clock[Employee Clock In/Out]
                Track[System Calculates Work Hours]
                Clock --> Track
            end

            subgraph Leave Management
                LeaveReq[Employee Requests Leave]
                LeaveApp[Manager/HR Approves Leave]
                LeaveReq --> LeaveApp
            end

            subgraph End of Month Processing
                AggTime[System Aggregates Total Hours & Deducts Leaves]
                GlobalSet[Fetch Global Settings: Base Hours, Tax Rate]
                GenPay[Finance Clicks 'Generate Payroll']
                AggTime --> GenPay
                GlobalSet --> GenPay
                LeaveApp --> AggTime
                Track --> AggTime
            end

            subgraph Payroll Finalization
                PayCalc[System Calculates Net Pay, Deductions, Overtime]
                ApprovePay[Finance Marks as 'Paid']
                PaySlip[Employee Downloads PDF Payslip]
                
                GenPay --> PayCalc
                PayCalc --> ApprovePay
                ApprovePay --> PaySlip
            end
            
            class Clock,LeaveReq,PaySlip emp
            class LeaveApp mgr
            class Track,AggTime,GlobalSet,PayCalc sys
            class GenPay,ApprovePay fin
